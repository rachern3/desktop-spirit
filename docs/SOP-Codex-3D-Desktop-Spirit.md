# 如何用 Codex 从 0 到做出一个 3D 桌面精灵

> 小白实操版：不要求会编程。你只需要准备参考图片、会截图、会描述哪里不满意，并按照本文把提示词一段段发给 Codex。

## 先说结论：小白能不能做？

能，但不是对 Codex 说一句“帮我做个桌宠”，然后等它一次性变出成品。

更现实的方式是：

1. 你负责告诉 Codex 想要什么。
2. Codex 负责分析、写代码、运行命令、检查文件和打包。
3. 遇到必须登录网页、上传模型、点击按钮的环节，你按 Codex 的指引操作。
4. 每完成一步，你看效果并反馈。
5. 一轮轮改到可以用，再让 Codex 生成正式版本。

整个过程很像你是产品负责人，Codex 是坐在旁边的设计师、程序员和测试人员。

## 最后能得到什么

按照本文做完，你会得到一个可以直接双击打开的桌面精灵：

- 透明背景，站在电脑桌面上。
- 可以用鼠标拖动位置。
- 可以转身查看正面、侧面和背面。
- 有待机、挥手、专注、伸展、充能等动作。
- 可以显示对话气泡。
- 可以接入本机 Codex 进行简单对话。
- 可以提醒 Codex 任务开始、等待确认和完成。
- 有退出、隐藏、置顶和鼠标穿透功能。

本文以 macOS 为例。Windows 也能做，但打包和托盘部分会有区别。

如果你想先动手、不想先读完整教程，可以直接跳到“第三部分：一套可以直接照抄的完整对话”，从对话 1 开始逐条发送；遇到问题再回到对应步骤看解释。

## 本项目 GitHub 仓库

本教程对应的完整项目已经公开在：

- [rachern3/desktop-spirit](https://github.com/rachern3/desktop-spirit)

如果你想先运行现有项目，再跟着教程修改，可以把仓库地址发给 Codex：

```text
这是项目仓库：https://github.com/rachern3/desktop-spirit
请帮我把项目下载到本地，安装依赖、运行检查并启动桌面精灵。
每一步用简单的话告诉我正在做什么；遇到需要登录或确认的操作时再提醒我。
```

已经配置 GitHub SSH 的电脑，也可以使用：

```bash
git clone git@github.com:rachern3/desktop-spirit.git
cd desktop-spirit
```

# 第一部分：先学会怎么和 Codex 说话

## 1. 不要把 Codex 当成搜索框

不够好的说法：

```text
帮我做个桌宠。
```

这句话没有说清楚角色、平台、操作、交付物和验收方式。Codex 只能自己猜，最后通常不是你想要的。

更好的说法：

```text
我想把附件里的角色做成一个 macOS 3D 桌面精灵。
我不会编程，请你负责创建项目、写代码、运行测试和打包。
我们分阶段完成，不要一次做完。
第一步先分析角色并做正面、侧面、背面设定图，给我确认后再开发。
每一步完成后告诉我去哪里看效果，以及我需要检查什么。
```

## 2. 万能提示词公式

以后每次让 Codex 做事，都可以套这个公式：

```text
我要什么：
参考什么：
这次只改什么：
哪些东西不要动：
完成后怎么让我检查：
```

例如：

```text
我要让翅膀更修长。
参考我刚上传的背面图片。
这次只改翅膀长度、宽度和中央机甲大小。
不要修改人物身材、脸、动作和窗口。
完成后给我看正面、45 度和背面三个角度，再打包。
```

## 3. 小白最重要的五条原则

### 原则一：一次只解决一个阶段

先做角色图，再做桌面原型，再做 3D，再加动作，最后接 AI。

如果同时要求“重新画角色、做 3D、加语音、接 AI、解决托盘”，出现问题后你不知道是哪一部分导致的。

### 原则二：先看效果，再让它打包

每次都说：

```text
先给我预览和检查结果，我确认后再生成正式版本。
```

### 原则三：问题要配截图

截图比“还是不对”“有点丑”更容易解决。

推荐这样说：

```text
这是现在的截图。
1. 人物看起来太矮胖。
2. 左手少了一截。
3. 侧面出现了多余的翅膀。
请先分析原因，只修这三个问题。
```

### 原则四：让 Codex 自己检查文件和运行测试

你不需要复制一堆代码。直接说：

```text
请你检查项目文件、运行测试、修复问题，并告诉我结果。
```

Codex 官方提供本地开发和终端能力，可以在获得权限的工作区中查看代码、执行构建和检查结果。参考：[Codex Developers](https://learn.chatgpt.com/docs/developers)。

### 原则五：每个满意阶段都要提交版本并留记录

```text
这个版本我满意。请先运行测试并检查 Git 状态，确认没有密码、密钥、node_modules、构建产物和个人配置将被提交。
检查通过后，把当前修改提交并推送到 GitHub，提交说明要写清这一阶段完成了什么。
最后告诉我提交编号、仓库链接、当前版本、正式文件位置、已知问题和下一步，并生成一个可恢复的检查点。
```

这样以后即使换了新任务，也可以通过 GitHub 找回之前的代码，不需要从头解释。

### 哪些时候应该提交 GitHub

建议在这些节点各提交一次：

1. 二维桌宠第一次可以运行。
2. 拖动、托盘、退出等桌面功能通过。
3. 3D 身体第一次接入成功。
4. 待机、挥手和转身动作通过。
5. 专注、伸展、充能和翅膀通过。
6. Codex 对话与进度提醒通过。
7. 正式版本打包完成。

每次可以直接对 Codex 说：

```text
这个阶段已经确认，请帮我保存到 GitHub：
1. 先运行测试；
2. 检查哪些文件会被提交；
3. 排除密码、密钥、依赖和打包产物；
4. 用一句容易看懂的话作为提交说明；
5. 提交并推送到当前 GitHub 仓库；
6. 完成后把提交编号和链接发给我。
```

如果这一版出现问题，可以说：

```text
请先查看 GitHub 提交记录，找到上一个确认过的版本。
告诉我这次修改和上一版的差异，先不要直接删除或覆盖文件，等我确认后再恢复。
```

# 第二部分：正式开始做桌面精灵

## 第 0 步：准备材料

你需要准备：

- 一张你喜欢的角色或手办照片。
- 一台 Mac。
- 已安装并登录的 Codex 桌面应用。
- 一个专门放项目的空文件夹。
- 愿意多截几次图、反馈几轮的耐心。

最好再准备：

- 正面照片。
- 侧面照片。
- 背面照片。
- 你喜欢的发型、翅膀和服装参考图。

图片越完整，后面的 3D 越稳定。

### 你要对 Codex 说

```text
我要在这个文件夹里开发一个 3D 桌面精灵。
我是小白，不会写代码。请你负责技术实现，并且每一步都用简单的话告诉我：
1. 你做了什么；
2. 我去哪里看；
3. 我需要点击什么；
4. 看到什么算成功。

先检查当前文件夹，不要删除我已有的文件。然后给项目建立清晰的 designs、work、outputs 和 release 目录。
```

### 看到什么算成功

Codex 会告诉你它创建或选择了哪个项目目录，并说明下一步需要参考图片。

如果它直接开始写很多代码，你可以打断：

```text
先不要开发。我们先确认角色外观和产品需求。
```

---

## 第 1 步：让 Codex 理解你的角色

把角色图片拖进 Codex，然后发送：

```text
以这个角色为原型做桌面精灵。
请先分析她的发型、脸、身材比例、服装、装甲、颜色、左右不对称部件和翅膀。
然后给我一份角色外观清单，并指出哪些地方在生成侧面和背面时最容易出错。
现在不要写程序。
```

### 你要重点确认

- 长发还是短发。
- 马尾在什么位置。
- 哪条腿有护甲。
- 翅膀从哪里长出来。
- 衣服主色是什么。
- 金属是银色、金色还是其他颜色。
- 角色是修长、可爱还是写实比例。

如果 Codex 理解错了，直接纠正：

```text
有三点需要纠正：
1. 她是高马尾，不是披肩发；
2. 左腿是银色护甲，右腿是白色长靴；
3. 翅膀是透明绿色晶体，不是羽毛。
请更新角色清单，其他内容不变。
```

### 为什么这一步不能省

如果角色设定没有锁定，后面经常会出现：

- 侧面多出一只手。
- 背面又出现一套翅膀。
- 左右腿装备互换。
- 同一个人正面修长、侧面却很矮胖。

---

## 第 2 步：先做角色全景图和 PSD

角色清单确认后，对 Codex 说：

```text
角色设定确认了。
请制作一张角色全景设定图，至少包含：正面、45 度、左侧面、背面和右侧面。
所有角度要保持同样的身高、脸、发型、服装和左右装备。
背景要方便抠图。
同时给我透明 PNG 和分层 PSD，方便后续继续修改。
先给我预览，不要直接进入桌宠开发。
```

### 你要怎么验收

把图片放大，逐项看：

- 每个角度是不是同一个人。
- 头顶和脚底有没有对齐。
- 手脚数量是否正确。
- 左右腿装备是否一致。
- 侧面有没有混进正面的手臂或翅膀。
- 背面的头发和翅膀是否合理。

### 不满意时怎么说

不要要求整张重做，先说清问题：

```text
图一正面和背面看起来矮胖，左手缺了一截。
图二侧面出现了正面的翅膀，图三出现了多余的手。
请保持脸、服装和配色不变，只重新处理身材比例、左手和侧面遮挡关系。
修改后重新给我五视图预览。
```

### 什么时候进入下一步

当你觉得“即使现在拿去做海报也能接受”，再继续。

---

## 第 3 步：先做一个能运行的二维桌宠

这里不要急着上 3D。先验证桌宠是不是好用。

对 Codex 说：

```text
角色图确认了。现在先做一个可运行的 macOS 桌面精灵原型。
我不会写代码，请你直接创建项目并完成。

第一版必须做到：
1. 背景透明；
2. 角色始终在其他窗口上面；
3. 左键拖动角色可以移动；
4. 点击角色会出现一句陪伴提示；
5. 有对话输入框；
6. 有设置按钮；
7. 有明确的退出方法；
8. 关闭后再次打开能记住位置。

完成后请直接运行给我看，并告诉我怎么操作。先不要做 3D。
```

### 你要亲手试什么

1. 点一下角色。
2. 按住角色拖动。
3. 把角色拖到屏幕边缘。
4. 打开设置。
5. 找退出按钮。
6. 退出后重新打开。

### 发现只能点击不能拖动

直接说：

```text
现在只能点击，无法拖动，点击后只有提示语变化。
请检查点击和拖动是否冲突。修复后让我重新测试：短按触发对话，按住移动才拖动窗口。
```

### 拖到屏幕边缘卡住

```text
角色拖到屏幕边缘就卡住了。我希望可以继续拖进去，只保留一小部分方便找回。
请修复拖动边界，但重启后仍要能把完全丢失的窗口找回来。
```

### 找不到退出

```text
我找不到退出桌面精灵的地方。
请同时增加：设置里的退出按钮、菜单栏托盘退出和显示/隐藏快捷键。
完成后告诉我每个入口在哪里。
```

---

## 第 4 步：把桌面基础功能做完整

二维原型能拖动后，再说：

```text
基础原型可以用了。请继续补齐桌宠必需功能：
1. 菜单栏托盘图标；
2. 显示/隐藏；
3. 窗口置顶开关；
4. 鼠标点击穿透；
5. 穿透后的恢复快捷键；
6. 重置窗口位置；
7. 防止同时打开两个桌宠；
8. 多显示器切换后不要丢失窗口。

请逐项测试。不要只告诉我代码写好了，要让我实际操作验证。
```

### 托盘图标没看到

```text
我在菜单栏里没有看到托盘图标。
请检查 macOS 托盘图标尺寸、Retina 图片和 Template Image 设置。
如果图标可能被刘海遮住，请在桌宠设置里再增加一个“打开系统菜单”按钮。
```

### 总是打开旧版

```text
我容易打开之前的旧版本。
以后每次生成正式版时，请删除 release 文件夹里的旧版本，只保留一个最新版本，并在文件名里写版本号。
同时保证旧进程退出后才能打开新版。
```

这条要求以后每次打包都要重复。

---

## 第 5 步：告诉 Codex，你要升级成真 3D

二维桌宠好用后，对 Codex 说：

```text
桌面功能已经通过。现在我要把角色升级成真 3D：
1. 鼠标拖动可以连续 360 度转身；
2. 不再切换多张二维图片；
3. 挥手时要真正抬手，不是整个角色左右晃；
4. 动作时人物不能在窗口里乱跑；
5. 正面、侧面和背面不能出现重复手脚或翅膀。

请先调查适合小白、尽量免费或有试用额度的 3D 生成与自动绑骨方案。
先给我路线和需要我配合的步骤，不要直接接入低质量模型。
```

### 这时 Codex 应该告诉你什么

- 需要准备哪些角度图片。
- 用什么方法生成 3D 身体。
- 哪些步骤可以本地完成。
- 哪些步骤需要登录网页。
- 如何做自动绑骨。
- 如何下载动作。
- 如何判断模型合不合格。

如果 Codex 只给你很多网站名称，没有执行计划：

```text
请不要只列工具。按照“我点击什么、你处理什么、完成后检查什么”的顺序，给我一个最短可执行方案。优先使用免费或试用渠道。
```

## 第 6 步：生成 3D 身体

### 最关键的一句话

```text
人物身体和透明翅膀必须分开处理。先生成没有翅膀的人体，翅膀以后单独做。
```

这是为了避免 AI 把翅膀、头发、手臂粘成一团。

### 你要对 Codex 说

```text
请根据已经确认的角色图，准备适合 3D 重建的正面、左侧、右侧和背面图片。
先移除翅膀和绿色透明特效，但保留人物、头发、服装和固定装甲。
四张图必须同高、同宽、脚底对齐。
准备好后告诉我需要上传哪个文件。
```

### 如果需要登录第三方网站

Codex 可能会让你登录 3D 生成网站。这时：

1. 你自己登录。
2. 不要把密码发给 Codex。
3. 登录完成后说：

```text
我已经登录了，继续。请告诉我下一步点击哪里、上传哪个文件、每个选项怎么选。
```

如果页面和 Codex 描述不一样，直接截图：

```text
我现在看到的是这个页面，下一步怎么填？
```

### 模型生成后先别急着用

对 Codex 说：

```text
模型生成好了。请先做八个方向的截图检查，不要马上接入桌宠。
重点检查身材比例、脸、五指、马尾、护腿、额外手脚和装甲变形。
如果不合格，明确告诉我应该重生成还是可以修。
```

### 模型很丑时怎么说

```text
这个 3D 模型比二维版差很多：脸被磨平、身体变矮胖、手指和装甲细节丢失。
请停止接入这个模型，分析是输入图、生成模式还是面数导致的，并给我下一轮更稳的方案。
```

要敢于停止失败模型。不要因为已经等了很久，就继续给一个不合格模型绑骨。

---

## 第 7 步：让 Codex 帮你降低模型体积

高质量模型可能太大，不能长期放在桌面运行。

你只需要说：

```text
这个模型的外观可以接受。请制作一个适合桌宠实时运行的轻量版本。
保留脸、手指、装甲轮廓、马尾、贴图和 UV。
完成后对比正面、侧面和背面，告诉我优化前后的面数和文件大小。
```

### 看到什么算成功

- 模型文件明显变小。
- 人物没有再次变矮胖。
- 脸和手指没有完全糊掉。
- 侧面和背面没有多出东西。

你不需要理解“减面算法”是什么。你只负责看外观有没有变坏。

---

## 第 8 步：在 Mixamo 自动绑骨

这是整个流程中最需要你亲手点击的环节之一。

### 8.1 上传哪个文件

让 Codex准备好后，你会得到一个 ZIP、OBJ 或 FBX 文件。

如果你不确定，截图问：

```text
是上传这个文件吗？
```

不要凭文件名猜。

### 8.2 进入 Auto-Rigger 第一页

你会看到角色预览。检查：

- 人物正面朝前。
- 头朝上、脚朝下。
- 没有横躺或背对你。

如果正确，对 Codex 说：

```text
我已经进入 Auto-Rigger，角色正面朝前。现在可以点 Next 吗？
```

### 8.3 放五组标记

页面会让你拖动圆圈。放到：

- Chin：下巴。
- Wrists：左右手腕。
- Elbows：左右手肘。
- Knees：左右膝盖。
- Groin：两腿之间、骨盆下方。

打开 `Use Symmetry`，骨骼选择 `Standard Skeleton (65)`。

放完后截图问：

```text
我把标记放成这样了，位置对吗？请逐个告诉我是否需要调整。
```

不要怕多问。标记放错，后面所有动作都会变形。

### 8.4 绑骨完成后

旋转角色，看肩膀、手肘、手腕、膝盖是否自然。如果看起来正常：

```text
自动绑骨完成了。请告诉我基础模型下载页面每个选项怎么选。
```

一般会选择：

```text
Format: FBX Binary
Pose: T-pose
Skin: With Skin
```

下载完成后只说：

```text
基础模型下载完成，继续。
```

Codex 会检查下载文件并继续处理。

## 第 9 步：下载动作

基础骨骼只需一次。接下来在 Mixamo 搜索动作。

本项目使用了：

- `Breathing Idle`：呼吸待机。
- `Waving`：挥手。
- `Left Turn 90`：左转。
- `Right Turn 90`：右转。
- `Thinking`：专注。
- `Female Dance Pose`：伸展姿态。
- `Female Laying Pose`：充能姿态。

### 每选一个动作都这样问

```text
我现在选中了 Breathing Idle，这是你说的动作吗？右边参数怎么调？
```

或者直接发截图。

### 下载动作时怎么选

通常选择：

```text
Format: FBX Binary
Skin: Without Skin
Frames per Second: 30
Keyframe Reduction: none
```

`Trim` 左右两端都拖到最外面，保留完整动作。

### 挥手太快太大

你可以先在 Mixamo 调成：

```text
Energy: 15
Direction: 55
Overdrive: 0
Character Arm-Space: 45
Mirror: Off
```

如果还是快，告诉 Codex：

```text
动作文件下载好了，但挥手还是偏快。请在桌宠运行时把播放速度降低，并增加平滑进入和返回待机，不要让我重新下载。
```

### 每下载一个文件后

你只需说：

```text
下载完成，继续。
```

如果 Codex 不知道文件在哪里：

```text
请检查我的下载文件夹，按修改时间找到刚下载的 FBX，但不要删除其他文件。
```

## 第 10 步：把技术处理交给 Codex

动作都下载好后，对 Codex 说：

```text
基础模型和所有动作都下载完成。
现在请你负责完成后续技术处理：
1. 把 FBX 转成桌宠能使用的 GLB；
2. 恢复人物颜色和 PBR 贴图；
3. 修复 UV 方向；
4. 去掉动作导致的人物位移；
5. 校准左右转身到准确 90 度；
6. 合并所有动作；
7. 每加入一个动作就检查一次，不要一次全部合并后才测试。

完成后告诉我最终模型在哪里，并生成正面、侧面、背面和动作截图给我检查。
```

### 你不需要做什么

- 不用自己打开代码编辑器。
- 不用自己修改模型文件。
- 不用理解 FBX、GLB、UV、PBR 的全部原理。
- 不用复制 Codex 输出的大段程序。

如果 Codex 需要你批准运行命令，确认命令是在项目目录中处理模型、不会删除个人文件后再批准。

### 模型没有颜色

```text
3D 人物现在是白色的，没有原来的颜色。
请检查是否丢失了基础色、法线和金属粗糙度贴图，以及 UV 是否翻转。
修复后给我同一角度的前后对比。
```

### 动作时人物会移动

```text
做动作时人物在窗口里发生位移，动作结束后位置也不稳定。
请检查 Hips 或 Root 的位移轨道，把动作处理为原地动作，同时保留自然的上下重心变化。
```

## 第 11 步：接入桌宠并逐个验收动作

对 Codex 说：

```text
模型检查通过。现在把它接入桌面精灵。
请先只接入呼吸待机、挥手和鼠标 360 度转身。
完成后运行给我测试。三项通过后，再接专注、伸展和充能。
```

为什么要分两次？因为如果六个动作一起出问题，很难判断是哪一步造成的。

### 你要测试的动作顺序

1. 普通站立 30 秒。
2. 挥手。
3. 左右拖动转身。
4. 专注。
5. 伸展。
6. 充能。
7. 每个动作结束后是否自然回到站立。

### 普通站姿不好看

```text
普通站立时人物有点后仰，眼睛也没有看着我，缺乏陪伴感。
请只调整待机站姿、头部和眼睛的轻微跟随，不要修改身材和动作文件。
```

### 伸展太高，手被截掉

```text
伸展时手被窗口顶部截掉。
请在伸展动作期间稍微缩小并下移人物，动作结束后平滑恢复，不要扩大整个桌宠窗口。
```

### 充能横向太长

```text
充能侧卧姿态横向太长。请把人物缩小一点并沿对角线倾斜摆放，让她完整显示在当前窗口里。
```

### 动作结束会卡一下

```text
充能结束回到待机时会卡一下。
请检查动作结尾和待机姿态的差异，增加平滑过渡，不要直接瞬间切换。
```

### 专注时头发偏移

```text
做专注动作时头顶头发和装饰偏了。
请稳定头部相关轨道，优先保持脸和头发位置；不要让整个人物僵住。
```

## 第 12 步：单独处理翅膀

翅膀是最适合单独迭代的部分。

### 第一次提出需求

```text
现在只处理翅膀。
参考我上传的背面图片，把翅膀做成透明绿色晶体，连接在背部中央机甲上。
不要把翅膀融合进人物身体。
先做 3D 版本，并给我正面、45 度、侧面和背面预览。
```

### 翅膀不好看时，不要只说“丑”

使用这个模板：

```text
翅膀需要继续调整：
1. 左右各减少一片；
2. 每片更修长；
3. 厚度减少，不要像厚塑料；
4. 中央机甲缩小；
5. 下方弯翼要和上面主翼连在一起。

只修改翅膀，不要动人物和动作。完成后给我 0、45、180 度预览。
```

### 如果想尝试贴图翅膀

可以说：

```text
我想测试直接使用这张翅膀贴图，不做完整 3D。请先做一个实验预览，不要替换正式版本。
重点检查连续转身时是否会跳变。
```

如果切换效果不好：

```text
二维翅膀转身时切换明显，放弃这个实验，恢复到上一版 3D 晶翼。不要保留角度贴图切换代码。
```

这就是为什么实验必须先预览、不要直接覆盖正式版。

## 第 13 步：让桌宠连接本地 Codex

这里分成两个功能。

### 功能一：和桌宠聊天

```text
请让桌宠优先调用我本机已经登录的 Codex 来回答对话。
要求：
1. 不让我把密钥写进前端；
2. 不读取当前项目文件；
3. 使用临时、只读的独立会话；
4. 本地 Codex 不可用时回退到离线回复；
5. 回复失败时给普通人能看懂的提示。

完成后请用三条问题测试，并告诉我它和当前 Codex 开发任务是否共享上下文。
```

要注意：桌宠对话通常是一个独立 Codex 会话，不等于直接接管你正在进行的开发任务。

### 功能二：提醒当前 Codex 任务进度

```text
我还想让桌宠提醒 Codex 任务进度。
请使用 Codex 生命周期 Hooks，实现：
1. 任务开始；
2. 正在执行；
3. 等待我确认；
4. 任务完成；
5. 失败。

只传任务标题、简短状态、时间和估算进度，不读取完整对话内容。
完成后告诉我需要信任或启用什么，并实际触发一次测试。
```

Codex 官方 Hooks 可以在任务生命周期事件出现时执行自定义命令。项目级 Hook 需要项目被信任。参考：[Codex Hooks](https://learn.chatgpt.com/docs/hooks)。

### 为什么百分比可能不准

Codex 并不知道任务还剩“精确 37%”。桌宠显示的百分比只是根据开始、执行工具、等待确认、完成等事件估算。

所以比起盯着百分比，更应该看：正在做、等你确认、已完成、失败。

## 第 14 步：让 Codex 做最终测试

当外观和功能都满意后，复制这段：

```text
准备生成正式版本。请先不要打包，完成一轮最终验收：

交互：
1. 点击；
2. 左键拖动移动；
3. 右键拖动转身；
4. 屏幕边缘拖动；
5. 点击穿透和恢复；
6. 托盘菜单；
7. 退出；
8. 防止多开。

外观：
1. 0、45、90、135、180、225、270、315 度；
2. 待机、挥手、专注、伸展、充能；
3. 每个动作开始、中间、结束；
4. 检查人物位移、穿模、手脚、头发、翅膀和窗口截断。

工程：
1. 运行全部测试；
2. 正式构建；
3. 确认版本号；
4. 告诉我仍有哪些已知限制。

发现问题先修复并重新验证，全部通过后再打包。
```

### 为什么一定要这样说

“能打开”不等于“做完了”。很多问题只会出现在侧面、动作中间、第二次启动或点击穿透后。

## 第 15 步：生成唯一正式版本

最终说：

```text
验收通过。请生成新的正式版本。
要求：
1. 版本号递增；
2. 删除 release 文件夹里的旧版本；
3. 只保留一个最新应用；
4. 检查应用内部版本号和文件名一致；

```

1. 先从设置或托盘退出旧桌宠。
2. 如果旧版没有退出入口，让 Codex 帮你检查并结束旧进程。
3. 再双击 release 文件夹中唯一的新版。

macOS 未签名开发版首次打开时，可能需要在 Finder 里右键应用，选择“打开”。

我要把附件角色做成 macOS 3D 桌面精灵。我是小白，请你负责开发，每一步告诉我去哪里看和怎么验收。先建立项目结构，不要直接写完整产品。

角色清单确认。请制作正面、45 度、侧面、背面、右侧面设定图，保持同一人物和同一比例，同时输出透明 PNG 和分层 PSD。先给我预览。

设定图通过。请先做透明、可拖动、始终置顶、可点击对话、可退出并能记住位置的二维桌宠原型。完成后直接运行让我测试。

请准备无翅膀人体四视图并生成 3D 身体。模型完成后先做八方向检查，不合格就停止，不要直接接入桌宠。

全部下载完成。请自动完成 FBX 转 GLB、PBR 贴图、UV 修复、去位移、转身校准、动作合并和逐段测试。给我最终模型和预览。

# 第四部分：遇到问题时该怎么继续说

## Codex 好像停住了

```text
请告诉我当前执行到哪一步、已经完成什么、正在等待什么，以及下一步是否需要我操作。不要重新开始已经完成的工作。
```

## Codex 一直说“继续”，但没有结果

```text
请不要只回复计划。现在检查实际文件和运行状态，继续执行到可以让我看到结果；如果被阻塞，明确告诉我卡在哪个文件、网页或权限。
```

## 双击命令文件没反应

```text
我双击这个 command 文件没有反应。请你检查文件权限、脚本路径和错误日志，并给我一个可以直接运行的替代方式。不要让我盲猜终端命令。
```

## 出现系统弹窗

截图后说：

```text
出现了这个系统弹窗。请解释它是什么意思、是否安全、我应该点哪个按钮，以及是否需要你修复文件权限或路径。
```

## 不知道上传哪个文件

```text
请直接告诉我完整文件名和所在文件夹，并说明页面里应该选择它还是选择 ZIP。不要只说“上传模型”。
```

## 不知道页面怎么填

```text
这是我当前页面的截图。请按从上到下的顺序告诉我每个选项怎么选，最后哪个按钮可以点。
```

## 改完反而更差

```text
这一版比上一版差。请先恢复到上一个确认过的版本，再分析这次修改为什么造成回退。恢复后只重新尝试一个变量，不要同时改很多地方。
```

## 不确定是不是新版

```text
请检查当前正在运行的进程、应用文件路径和内部版本号，退出旧版后只打开 release 文件夹里的唯一新版。
```

## 让 Codex少讲技术，多做事

```text
我不需要技术原理。请直接执行你能执行的部分；只有必须由我登录、上传、点击或确认时，再用简单步骤指导我。
```

# 第五部分：哪些事 Codex 能做，哪些事需要你做

## Codex 适合负责

- 创建和整理项目文件。
- 编写和修改程序。
- 运行测试与构建。
- 分析截图和错误日志。
- 处理 PNG、PSD、GLB、FBX 等资产流水线。
- 检查模型面数、动画、根位移和贴图。
- 启动本地预览。
- 生成应用包。
- 记录进度和生成文档。

## 通常需要你亲手做

- 登录第三方网站。
- 输入密码、验证码。
- 同意第三方服务条款。
- 上传模型到你自己的账号。
- 在 Mixamo 页面拖动关节标记。
- 观察角色是否好看。
- 决定哪个方案更符合你的审美。
- 对外发布前确认素材版权和服务条款。

## 不应该交给任何提示词的内容

- 不要把账号密码发进对话。
- 不要把 API Key 写在截图或前端代码里。
- 不要授权删除整个个人目录。
- 不要在不理解目标的情况下批准处理工作区外大量文件。

# 第六部分：小白验收清单

## 角色外观

- [ ] 正面、侧面、背面是同一个人。
- [ ] 手脚数量正确。
- [ ] 左右装备没有互换。
- [ ] 身材没有变矮胖。
- [ ] 头发和脸没有严重变形。
- [ ] 翅膀没有重复或突然切换。

## 动作

- [ ] 普通站立自然。
- [ ] 眼睛和头部有轻微关注感。
- [ ] 挥手是真的手臂动作。
- [ ] 转身时脚步和朝向一致。
- [ ] 专注时头发不乱跑。
- [ ] 伸展时手不会被截掉。
- [ ] 充能姿态不会超出窗口。
- [ ] 动作结束不会卡住或瞬间跳回。
- [ ] 人物不会在窗口里累计位移。

## 桌面功能

- [ ] 可以点击。
- [ ] 可以拖动。

# 最后记住一件事

你不需要先学会 React、Electron、Three.js、GLB 和骨骼动画，才能开始做桌面精灵。

你真正需要学会的是：

- 把一个大目标拆成小步骤。
- 每一步都告诉 Codex 什么不能变。
- 让 Codex 先执行和预览，再打包。
- 用截图指出具体问题。
- 对满意版本及时做检查点。

只要你能判断“这是不是我想要的”，并能说清楚“哪里不对”，就已经具备和 Codex 合作做出成品最重要的能力。
